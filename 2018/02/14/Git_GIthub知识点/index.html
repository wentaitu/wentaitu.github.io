问题

1.git clone 或者 git push时的错误信息：
fatal: unable to access 'https://github.com/Anbeautifulday/SocialMotion.git/': Failed to connect to 127.0.0.1 port 42097: 拒绝连接
解决时，终端输入如下两行命令:
unset https_proxy
unset HTTPS_PROXY
未解决:每次git clone 或者 git push都需要输入如上两条命令?

2.Github官网更改用户名后
主页由https://github.com/anbeautifulday变为https://github.com/wentaitu
官网提示
Your account has been renamed.
Remember to update your local repositories to point to the location.
https://help.github.com/articles/changing-a-remote-s-url/#platform-linux

比如在原来项目下使用git pull则路径不对
已测试:
在对应项目目录下的终端中输入git config -e
更改为对应地址ctrl+O保存,ctrl+Z退出即可

3.git pull之后有冲突,先git commit -a -m ‘说明’
然后再git pull,打开ANdroid studio选中标红文件名右键Git->Resolve Conflicts->Merge
a、左边的Local Changes代表”当前“分支上的修改；
b、右边的Changes from Server代表“合并进来”的分支上的修改；
c、中间的Result代表经过处理后的最终内容；
红色区域：代表当前分支和合并分支都编辑过的内容，属于冲突；
蓝色区域：代表被单方面编辑过的内容，属于改变；
灰色区域：代表被删除的内容，属于改变；
绿色区域：代表新增的内容，属于改变；
可参考:http://blog.csdn.net/jaden_hool/article/details/51852196
