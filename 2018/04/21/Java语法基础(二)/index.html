1.== 与equals()

在C++中，两个字符串比较的代码可以为：
(string1==string2)
但在java中，这个代码即使在两个字符串完全相同的情况下也会返回false
Java中必须使用string1.equals(string2)来进行判断
string s1="Hello";
string s2="Hello";
则(s1==s2)=true;
因为他们指向的同一个对象。在java中字符串的值是不可改变的，相同的字符串在内存中只会存一份，所以s1和s2指向的是同一个对象；
String s1=new String("Hello");
String s2=new String("Hello");
则(s1==s2)=false
如果把其他变量的值赋给s1和s2，即使内容相同，由于不是指向同一个对象，也会返回false。所以建议使用equals()，因为equals比较的才是真正的内容
例如：
String string1=new String( “aaa” );
String string2=new String( “aaa” );
这两个字符串当然应该是相等的。
如果用表达式string1==string2，则该表达式的值为false
如果用表达式string1.equals(string2)，则该表达式的值为true
因此应该用string1.equals(string2)，在if语句中就是
if(string1.equals(string2)) {

string1==string2,是值相等,而且内存地址也相等,是完全的相等???
string1.equals(string2)为true,只是值相等

问题描述：方法a处于不停执行状态，使得方法a中的一个整型数据num不断增加。若在方法a中当num > 100时则输出语句“SUCCESS！”，将会一直输出，若更改条件为num==100数据增加又可能从99跳到101,如何使得“SUCCESS！”只输出一次？

解决：设置bool变量status = false，若num > 100 则status = true, 之后外部控制，若status == true 则输出“SUCCESS！”

那又如何进行外部控制呢？设置另一变量bool变量another = true

故总结为
if (num > 100)
{
status = true;
If (status == true && another == true)
{
cout << “SUCCESS”;
}
another = false;
}

6.Java子类重写父类方法

方法的重写遵循两同两小一大
两同：方法名和参数列表相同(签名相同)
两小：子类返回值类型和抛出异常小于等于父类(如果返回值是void基本类型,那么重写方法必须是void,子类方法返回类型必须是父类方法返回类型的子类)
一大：子类访问控制权限大于等于父类

7.实例化子类对象自动调用父类构造方法

调用构造方法首先是顶级父类,然后是上一级父类,最后是子类.
故,实例化子类对象时首先要实例化父类对象,然后再实例化子类对象,所以在子类构造方法访问父类构造方法前,父类已经完成初始化操作
实例化子类对象时,父类无参构造方法将被自动调用,但有参构造方法并不能被自动调用,只能依赖于super关键字显示调用父类构造方法.如super(“test”, 123) 初始化继承下的父类的私有成员
