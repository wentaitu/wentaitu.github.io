物联网测试-OneNET

Behind every successful man there’s a lot u unsuccessful years.

传输协议：HTTP
硬件：麒麟开发板V2.4，M6311模块和移动卡完成网络连接
云平台：OneNET
服务器配置：LAMP
开发板代码

测试传感器数据通过M6311传输至OneNET并推送到自己服务器数据库

1. DTU和OneNET平台交互原理解析

  • 设备启动后，DTU上电自检完成硬件初始化、RTOS初始化，MCU为M6311模块配置一个串口作为与M6311模块数据交互的通道。MCU将给M6311模块发送一个使能信号，启动M6311模块，M6311模块完成SIM卡的读取，并连接移动网络。
  • 接着MCU将数据帧取出，将数据帧以HTTP API指定的数据包形式进行封装，通过AT指令控制M6311与对应服务器和端口建立TCP连接，进行数据透传，实现DTU的数据传输至OneNET.

2. OneNET平台数据传输至指定云服务器

  • 利用OneNET提供的数据推送服务，在接收到项目相关设备的数据时， OneNET将以HTTP POST请求形式向第三方平台的URL推送数据，推送数据相关信息以JSON串的形式置于HTTP请求中的body部分, 将数据推送到指定的接收地址。服务器后台接收到对应的JSON数据，先对内容进行解析，根据数据流名称的不同，分别存入数据库不同的表中，即一个数据流对应一个表，完成数据流的分类，便于后续展示平台的操作。以阿里云服务器为例，建立MySQL数据库，使其作为数据的最终存储单元。利用OneNET提供的第三方开发平台完成数据发送至数据库并完成存储。

    发送测试：

  1. 云服务器建立数据库，并建立相应表和字段
    CREATE TABLE 'Reco' ('temp' int(2) NOT NULL );
  2. OneNET上进行相应第三方开发平台配置
    （1）设置HTTP请求地址
    （2）设置Token,用于接入验证
    （3）采用消息加密，生成EncodingAESKey
  3. 建立服务器后台，完成数据推送
